Apis

Application Programming Interface - Interfaz de Programación de Aplicaciones

Una API es un conjunto de reglas predeterminadas que nos van a permitir que se comuniquen dos sistemas diferentes

Se utiliza mucho cuando queremos que un frotend consuma un backend, se comunican entre sí a través de una API

Las APIs funcionan comi intermediarios que permiten que un software solicite servicios o datos de otro software

Formatos y protocolos

REST (Representational State Transfer)

REST es el estilo arquitectónico más común para APIs web modernas. Sus características principales son:

  • Arquitectura cliente-servidor: Separa las responsabilidades entre quien solicita los recursos (cliente) y quien los proporciona (servidor).
  • Sin estado (Stateless): Cada solicitud del cliente contiene toda la información necesaria, sin depender de solicitudes anteriores.
  • Operaciones CRUD mediante métodos HTTP:
    • GET (obtener recursos)
    • POST (crear recursos)
    • PUT/PATCH (actualizar recursos)
    • DELETE (eliminar recursos)
  • Recursos identificados por URLs: Cada recurso tiene su propia dirección web única (por ejemplo, https://api.ejemplo.com/usuarios/123).

GraphQL

GraphQL es una alternativa más reciente a REST, desarrollada por Facebook:

  • Consultas flexibles: El cliente especifica exactamente qué datos necesita, evitando transferencias excesivas de información.
  • Punto de entrada único: A diferencia de REST que tiene múltiples endpoints, GraphQL típicamente usa un solo endpoint.
  • Operaciones: Query (consultar), Mutation (modificar) y Subscription (recibir actualizaciones en tiempo real).
  • Schema tipado: Define claramente la estructura de datos disponibles y las operaciones permitidas.

Formatos de Datos

JSON (JavaScript Object Notation)

  • El formato más popular actualmente
  • Ligero y fácil de leer por humanos
  • Basado en pares clave-valor: {"nombre": "Juan", "edad": 30}
  • Compatible naturalmente con JavaScript y soportado por la mayoría de lenguajes

XML (eXtensible Markup Language)

  • Formato más antiguo pero aún utilizado
  • Utiliza etiquetas similares a HTML: <usuario><nombre>Juan</nombre><edad>30</edad></usuario>
  • Más verboso que JSON pero ofrece validación mediante esquemas (XSD)
  • Común en servicios web SOAP y sistemas empresariales antiguos

Protocolos de Comunicación

  • HTTP/HTTPS: El protocolo estándar para transferencia de datos web
  • WebSockets: Permite comunicación bidireccional en tiempo real
  • gRPC: Sistema de Google para comunicaciones rápidas entre microservicios

Autenticación y Seguridad

Las APIs modernas implementan varios métodos de autenticación:

  • API Keys: Claves simples incluidas en las solicitudes
  • OAuth 2.0: Estándar para autorización delegada
  • JWT (JSON Web Tokens): Tokens firmados para transmitir identidad
  • HTTPS: Cifrado de datos durante la transmisión
